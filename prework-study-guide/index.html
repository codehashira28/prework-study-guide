HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Semantic elements are HTML elements that clearly describe their meaning to both the browser and other developers.
  • We use semanctic HTML elements to organize the content of our page.
  • Semantic elements are also good for SEO (search engine optimization) and accessibilty (i.e. screen readers).
  • Examples of semantic elements include: <header>, <footer>, <nav>, <aside>, <article>, and <section> because they clearly define their content and purpose.
  • Non-semantic element examples are the div and span elements because they tell us nothing about their content.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS color codes in hex, rgb, and hsl can be found here
  • Class attribute selectors are designated with a '.' in front of the class name

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stage your changes for commit

JavaScript

    • A variable is a named container that allows us to store data in our code.
    • Control flow is the order in which a computer executes code in a script.
    • We can introduce decision making in our code using if/else statements.
    • Arrays can be used to store related data as a list in a single variable.
    • Loops in our code can be used to repeat code over and over again.
    • Functions are blocks of reusable code that perform a specific task and can be called anywhere in our JavaScript file.
